# Automated Route Breakdown Dashboard

This project is a public, anonymised version of an Excel-based operational reporting tool designed to summarise route and booking performance for event transport services.

The original workflow involved sales and operational data that needed to be converted into a clear breakdown of events, monthly activity, order counts, revenue indicators and route/service performance. The public version uses anonymised event names, generic locations, month-level dates and transformed revenue values to protect confidential company data while preserving the structure of the analysis.

## Project Objective

The goal was to create a reporting tool that helps operations and business users quickly understand:

- Which event services generated the highest activity
- How monthly order and revenue trends changed over time
- How route/service demand varied across the reporting period
- Which areas required closer review for planning and operational decisions

## Key Features

- Clean public dashboard built from anonymised operational data
- Event names replaced with generic labels such as `Event 001`, `Event 002`
- Dates shown at month level only
- Order counts retained
- Revenue shown with generic `$` formatting and transformed values
- Confidential company, artist, venue, route and pickup-location details removed
- Dashboard designed for quick operational review

## Data Privacy

This project uses a public-safe version of the original workbook.

To protect confidentiality:

- Company identifiers were removed
- Event, route, artist, venue and pickup details were anonymised
- Exact dates were reduced to month-level reporting
- Revenue values were transformed
- No customer-level personal data is included
